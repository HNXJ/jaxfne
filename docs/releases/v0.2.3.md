# jaxfne v0.2.3 — Public Example Output Integration + Single-Neuron Tutorial

**Released:** 2026-05-19  
**Source:** dev @ 1305cb9  
**Tag:** `v0.2.3`

---

## Summary

v0.2.3 bundles three user-facing improvements over v0.2.1:

1. **Public Example Output Integration (v0.2.2):** Example 02 (spectrolaminar oddball scaffold) now emits standardized `probe_report.json` using the v0.2.1 multimodal operator contract.
2. **API Compatibility Fix (v0.2.2):** Added `ReadoutResult.name` alias (maps to `spec_name`) to fix public README example patterns.
3. **Single-Neuron Educational Tutorial (v0.2.3):** New example, documentation, and validation tests demonstrating all eight proxy operators on a minimal Izhikevich neuron.

---

## Features & Fixes

### v0.2.2: Public Example Integration

- **examples/02_spectrolaminar_oddball_scaffold.py:** Now generates `probe_report.json` with all eight operators:
  - Spike readout (`spk`)
  - Membrane voltage (`vm`)
  - Source proxy (`source`)
  - LFP-proxy, CSD-proxy, EEG-proxy, MEG-proxy, EMM-proxy

- **Output bundle structure:** Each example now emits:
  ```
  outputs/v0XY_scenario_name/
  ├── manifest.json                (model/field metadata)
  ├── metrics.json                 (windowed spectrolaminar metrics)
  ├── objective_report.json        (objective evaluation)
  ├── validation_report.json       (validation metadata audit)
  ├── probe_report.json            (multimodal readouts)
  └── asset_hashes.json            (file integrity)
  ```

### v0.2.2: API Compatibility

- **ReadoutResult.name alias:** Public examples use `result.name` in iteration patterns:
  ```python
  for result in results:
      print(result.name, result.metric, result.value, result.status)
  ```
  The alias maps to the internal `spec_name` attribute, preserving backwards compatibility.

### v0.2.2: Proxy/Conservation Metadata

- Enhanced field validation metadata in `validate_projection_invariants()`:
  - `source_current_conservation_status`: `"not_applicable_proxy_mode"` (clarifies no physical conservation is solved)
  - `boundary_condition_status`: `"declared_metadata_only"` (no PDE boundary solve)
  - `gauge_status`: `"declared_metadata_only"` (laminar proxy, not gauge-fixed)
  - `kernel_normalization_definition`: `"contact_rows_sum_to_one_proxy"` (normalization, not physics)

### v0.2.3: Single-Neuron Multimodal Tutorial

- **examples/03_single_neuron_multimodal_probe.py** (375 lines)
  - Single excitatory neuron, 100 ms CPU-safe simulation
  - Demonstrates all eight proxy operators in sequence
  - Generates complete output bundle with standardized manifest, metrics, validation, and probe reports
  - Educational comments explain claim-status metadata and operator status

- **docs/tutorials/01_single_neuron_multimodal.md** (289 lines)
  - User-facing documentation with running instructions
  - Detailed operator explanations and validation metadata reference
  - Adaptation guide for custom configurations
  - Scientific truth-status disclaimer

- **tests/test_single_neuron_multimodal_v023.py** (262 lines)
  - 14 test cases validating tutorial contract
  - Ensures output bundle completeness and JSON strictness
  - Validates all eight readout operators and claim-status metadata
  - Asserts EMM-proxy non-biological-metabolism status

## Claim-Status Metadata (Frozen)

All validation metadata remains unchanged from v0.2.1. Machine-readable fields:

- `truth_mode`: `truth_safe_unverified`
- `claim_level`: `computational_scaffold`
- `source_calibration_status`: `uncalibrated_izhikevich_native_current`
- `source_projection_mode`: `proxy_no_field_solve`
- `field_solver_status`: `laminar_proxy_no_pde`
- `field_claim_level`: `proxy_readout_only`
- `physical_amplitude_claim_allowed`: `false`
- `empirical_validation_status`: `not_empirically_validated`
- `mechanism_claim_status`: `not_claimed`

**Meaning:** This release is a computational scaffold for reproducible demonstrations. No biological/physical claims. No empirical validation. Claim-status metadata remains frozen across all operators.

## What This Release Is

- ✅ A validated computational scaffold for TFNE forward-field simulations
- ✅ Educational examples demonstrating multimodal operator composition
- ✅ Standardized JSON output bundles for reproducible documentation

## What This Release Is NOT

- ❌ Biological/physical validation (no empirical calibration)
- ❌ Whole-brain simulation (laminar column only)
- ❌ Field solver (proxy readout only; no PDE solve)
- ❌ Source conservation guarantee (proxy mode explicitly marked)
